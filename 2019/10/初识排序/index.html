举个栗子

    主人公可爱班里有5个人，5个人分别考了5分，3分，5分，2分和8分，现在需要给他们的
    分数排个序，咋搞嘞，这个时候我们可以借助数组来帮忙

  • 首先创建一个数组

  • 接下来就是代码

    #include<stdio.h>

int main()
{
	int a[11],i,j,l;
	int num=0;
	for(i=0;i<=10;i++)	//这个循环就是给我们创建的数组赋个初值，把它都变成0，因为数组的
		a[i]=0;			//值是用来计数的
		
	for(i=1;i<=5;i++)	//这个循环好理解，输入5个你要排序的数，注意哦，这个数不可以
	{					//大于9，原因看看我画的图你就知道了，数组的下标是分数的大小
		scanf("%d",&l);
		a[l]++;
	}
	
	for(i=0;i<=10;i++)		//这个循环就是把数组从下标0-9，输出，比如a[1]=2，说明，有两个的分数都是1
	{						
		for(j=1;j<=a[i];j++)
		{	
			printf("%d ",i);
			num++;
		}
		num++;
	}
	printf("\n%d\n",num);	//这个是我用来测量时间复杂度的，下面会看到
	getchar();		//这个getcahr(),可有可无，如果你是在ide运行这段代码，那加不加无所谓，如果是写在记事本
	return 0;		//用cmd或者powershell等终端运行，那就必须得加，原因你们懂的
}

总结一下

    这篇文章讲的其实是简化版的桶排序。桶排序从1956年就开始被使用，该算法思想是由`E.J Issac`和
    `R.C Singleton`提出来的，而且这个简化版的桶排序相当快哦。第7行循环m（m为桶的个数）次，第10
    行循环了n（n为待排序个数）次，第16，17行循环了m+n次（也就是变量num），所以这个时间复杂度为
    O（m+n+m+n）=O(2*(m+n)),这里告诉大家，我们说时间复杂度的时候一般忽略较小的常熟，所以桶排序
    最终复杂度为O（m+n），所以相当快。

    这个虽然有排序功能，但这个并不是实际意义的排序，为啥呢？因为实际生活中，考试分数不可能只有9
    分一般都是几十几百，那这个时候我们就需要创建a[1-1000]这样大的数组，很明显不合理

    而且实际生活中成绩得跟姓名等信息挂钩，如果这样，这个排序就无法实现，所以我们需要学习新的
    排序方法，我们的`冒泡排序`就出来了
