基本思想

冒泡排序是所有排序算法中最通俗易懂的，基本思想：是每次比较两个相邻的元素，如果他们的顺序错误
就把他们交换过来。

核心代码

  • 1、第一步先了解冒泡排序核心算法的大概

    冒泡排序的原理是：每一趟只能确定将一个数归位。即第一趟（大循环）只能确定将末位上的数归位，第
二趟只能将倒数第2位上的数归位，以此类推，跟冒泡泡一样。了解这个之后请看下列代码

  • 2、让我来看一下核心代码的一些细节及注意事项

全部代码

#include<iostream>
using namespace std;

int main()
{
	int i = NULL;
	int arr[] = { 23,94,-23,9,84,8,19,-3,8,4 };
	for ( i = 0; i < 10; i++)
	{
		for (int j = 0; j < (10 - i); j++)
		{
			if (arr[j] < arr[j + 1])
			{
				int t = NULL;
				t = arr[j];
				arr[j] = arr[j + 1];
				arr[j + 1] = t;
			}
		}
	}
	
	cout << "冒泡的结果为：" << endl;
	for (i = 0; i < 10; i++)
	{
		cout << arr[i] << "\t";
	}
	getchar();
	return 0;
}

总结一下

基本上大家看到冒泡排序第一反应就是————双重嵌套循环，通过上次的桶排序大家应该很快知道冒泡排序正常
情况下，时间复杂度是T(N)=O(n^2)，也就是双循环的次数。
这是一个非常高的时间复杂度，可能很多人没感觉，初学排序的小伙伴对这个n^2没啥感觉，对比之前的桶排序
好像是快点，但是具体快多少呢，我们可以测试一下冒泡的速度，这里采用库函数去测试，如果会最短路径的小
伙伴也可以自行测试哟！


* 看到了么，我们生成10000元素的数组，用随机函数生成10000个1000以内的数，只要几百毫秒，可能小伙伴 还是没什么感觉，这道底为啥说n^2很慢啊，这才几百毫秒么。等到我们讲完下面的排序大家就会恍然大悟，先卖个关子。
