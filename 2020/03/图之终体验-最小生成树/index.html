前提提要

最小生成树是一个给定的无向图G(V,E)中求一棵树，使得这棵树拥有无向图的所有顶点，且所有边都来自图G中的边，并且满足整棵树的边权之和最小。 看上图，其中橙色的线即为最小生成树的边，可以看到AB、BC、BD包含了图G的所有顶点，且边权之和为6，是所有生成树中权值最小的(例如边AD、BD、CD生成的树，其边权和为7).
那么最小生成树有3个性质要掌握:

  • 对给定的图G(V,E)，其最小生成树可以不唯一，但其边权之和一定唯一
  • 最小生成树是树(也就是一个一维数组)，因此其边数等于顶点数-1，且树内一定不会有环(看到环，直接想到并查集)

  • 最小生成树是在无向图上生成的，因此根节点可以是这棵树上任意一点，但是呢，如果题目说了要求最小生成树本身的值(比如说求边权之和)，为了让结果唯一,一般都会给出根节点(一般就是0号节点) 求最小生成树一般有两种策略，分别为prim算法和kruskal算法，这两个算法都是贪心的思想，只是贪心的策略不一样。用如下例题帮助大家理解

    题目

    亚历山大要讨伐恶魔大陆，但是恶魔大陆提前知道了亚历山大要来攻打，因此恶魔们事先对所有道路进行了冻结，希望以此消 耗亚历山大的体力去回复这些道路。但是亚历山大在分析恶魔大陆的地图之后打算从防守最薄弱的V0开始进攻，并且使用“爆 裂模式”来对抗他们，亚历山大可以随时回复任意一条已攻占城市所连接的道路，但是需要消耗那条道路的距离大小的体力（也 就是说，道路有多长，就需要消耗多少体力去回复）。并且在恢复某条路之后，亚历山大趁机攻占道路所连接的未攻占城市。 为了尽可能节省体力，它需要解决一个问题：如何选择需要恢复的道路，使得亚历山大可以消耗最少的体力，并保证他可以攻 占所有城市.

    样例输入

    第一行给出城市个数和道路数，第二行开始，每行依次输入i,j,k，i和j表示节点i-节点j，k表示节点i-节点j的路径长短。

    Input:

    6 10
0 1 4
0 4 1
0 5 2
1 2 6
1 5 3
2 3 6
2 5 5
3 4 4
3 5 5
4 5 3

    输出第一行是消耗的最少的体力

    Output:

    15

    prim算法(普里姆算法)

    prim算法其实跟dijkstra算法差不多，代码上唯一的差别的就是数组d的含义不同，那背后的含义就是，对于prim算法来说，数组d是一个集合S，存放了点到集合S最近的距离，dijkstra算法的数组d表示从起点到当前节点的最短路径数值，所以你要是会dijkstra算法，这个就很容易理解

    #include<iostream>
#include<cstdio>
#include<vector>
#include<cstring>
#include<sstream>
#include<algorithm>
#pragma warning(disable:4996)
using namespace std;

constexpr auto MAXN = 1000;
constexpr auto INF = 1000000000;

int n, m;
int G[MAXN][MAXN], d[MAXN];
bool vis[MAXN] = { false };
int prim();

int main()
{
#ifdef ONLINE_JUDGE                
#else    
	freopen("1.txt", "r", stdin);
#endif 

	cin >> n >> m;
	
	fill(G[0], G[0] + MAXN * MAXN, INF);
	int a1, a2, a3;
	for (int i = 0; i < m; i++)
	{
		cin >> a1 >> a2 >> a3;
		G[a1][a2] = G[a2][a1] = a3;
	}

	int ans = prim();

	cout << ans;


	system("pause");
	return 0;
}

int prim()
{
	int ans = 0;

	fill(d, d + MAXN, INF);
	d[0] = 0;
	for (int i = 0; i < n; i++)
	{
		int u = -1, min = INF;
		for (int j = 0; j < n; j++)
		{
			if (vis[j] == false && d[j] < min)
			{
				u = j;
				min = d[j];
			}
		}

		if (u == -1)	return -1;
		
		ans += d[u];
		vis[u] = true;
		for (int v = 0; v < n; v++)
		{
			if (vis[v] == false && G[u][v] != INF)
			{
            //注意这是跟dijkstra的区别，重点
				//以U为中介点可以使点v离集合s更近
				if (G[u][v] < d[v])
				{
					d[v] = G[u][v];
				}
			}
		}

	}
	return ans;
}

kruskal算法(克鲁斯卡尔算法)

这个算法的核心就是“边贪心”策略，在初始状态下隐去图G的所有边，这样每个顶点都是一个连通块，然后有如下3步操作：

  • 对所有边按边权从小到大进行排序
  • 按边权从小到大测试所有边，如果当前测试边所连接的两个顶点不在同一个连通块中，则把这条边加入当前最小生成树中；否则将边丢弃(看到这一句，应该立马想到并查集，继而想到并查集可以判断一个图中是否有环，符合最小生成树的要求，如果有环，就是将边丢弃)

  • 循环执行上面第二个步骤，知道最小生成树中的边数=总顶点-1 or 测试完所有边时结束(因为不是所有无向图都有最小生成树的哟)

    #include<iostream>
#include<cstdio>
#include<vector>
#include<cstring>
#include<sstream>
#include<algorithm>
#pragma warning(disable:4996)
using namespace std;

constexpr auto MAXN = 1000;
constexpr auto INF = 1000000000;

struct edge
{
	int u, v;
	int cost;
}E[MAXN];

bool cmp(edge a, edge b)
{
	return a.cost < b.cost;
}

int father[MAXN];
int findFather(int x)
{
	//int a = x;
	while (x != father[x])
		x = father[x];

	/*while (a != father[a])
	{
		int z = a;
		a = father[a];
		father[z] = x;
	}*/
	return x;
}

int kruskal(int n, int m)
{
	int ans = 0, Num_edge = 0;
	for (int i = 0; i < n; i++)
		father[i]=i;

	sort(E,E+m,cmp);

	for (int i = 0; i < m; i++)
	{
		int fau = findFather(E[i].u);
		int fav = findFather(E[i].v);
		if (fau != fav)
		{
			father[fau] = fav;
			ans += E[i].cost;
			Num_edge++;
			if (Num_edge == n - 1)
				break;
		}
	}

	if (Num_edge != n - 1)
		return -1;
	else
		return ans;

}
int main()
{
#ifdef ONLINE_JUDGE                
#else    
	freopen("1.txt", "r", stdin);
#endif 

	int n, m;
	cin >> n >> m;
	
	for (int i = 0; i < m; i++)
		cin >> E[i].u >> E[i].v >> E[i].cost;

	int ans = kruskal(n, m);

	cout << ans;

	system("pause");
	return 0;
}

总结

prim算法和kruskal算法肯定各有优缺点，如果你懂了就会发现，prim是通过顶点来形成最小生成树，而kruskal是通过遍历所有边来形成，所以prim算法适用于图G顶点数目较少而边数较多的情况下(即稠密图)，kruskal适合图G顶点数目较多而边数较少的情况下(即稀疏图)。如果还不理解的，可以私信我哟！
