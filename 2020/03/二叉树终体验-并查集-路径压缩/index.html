并查集

并查集，顾名思义就是“合并+“查”+“集合”有很多用途，比如说检查无向图中是否有环(kruskal算法就有用到)，也可以合并两个子集等

  • 并查集中的”并”和”查”函数，至于”集合”，就没有具体的代码了，所有的“集合”都存在father数组里面

    int findfather(int x)
{
while(x!=father[x])
    x=father[x];
return x;
}

void Union(int a,int b)
{
int fa=findfather(a);
int fb=findfather(b);
if(fa!=fb)
    father[fa]=fb;
}

路径压缩

如果你学会了并查集，你就应该知道，会有一种极端的情况让，如果合并的结果是有可能是一个线性结构或者是一个深度很大的数，那么这样的话在查找和后续的一些其他操作进行时会让时间复杂度大大提高，所以有了一个改进版并查集，也就是我们要介绍的第二个重点*路径压缩*。
路径压缩其实就是在查找的时候把所有节点的根节点都变成同一个，看图。
把上图的转变换成代码，就是在查找的时候顺带把当前节点的根节点改变一下，具体操作希望大家单步调式一下代码

int findfather(int x)
{
    int a=x;
    while(x!=father[x])
        x=father[x];
    
    //代码到这里，x就是根节点的下标，也就是当前你传回来的节点的根节点的下标
    //例如上图，你查找4号节点，那么x==1，懂了把，然后再看下面的代码，是如何把2，3号节点的父节点变成1的
    while(a!=father[a])
    {
        int z=a;
        a=father[a];
        father[z]=x;
    }
}

例题

以上如果会了呢，我们闲话不多说，上例题

有一个叫做“数码世界”奇异空间，在数码世界里生活着许许多多的数码宝贝，其中有些数码宝贝之间可能是好朋友，并且数码宝贝世界有两条不成文的规定： 第一，数码宝贝A和数码宝贝B是好朋友等价于数码宝贝B与数码宝贝A是好朋友 第二，如果数码宝贝A和数码宝贝C是好朋友，而数码宝贝B和数码宝贝C也是好朋友，那么A和B也是好朋友，现在给出这些数码宝贝中所有好朋友的信息，问：可以把这些数码宝贝分成多少组，满足每组中的任意两个数码宝贝都是好朋友，而且任意两组之间的数码宝贝都不是好朋友

样例输入

Input1:

4 2
1 4
2 3

Output1:

2

Input2:

7 5
1 2
2 3
3 1
1 4
5 6

Output2:

3

#include<iostream>
#include<cstdio>
#include<vector>
#include<cstring>
#include<sstream>
#include<algorithm>
#pragma warning(disable:4996)
using namespace std;

int father[101];
bool isroot[101];

int findfather(int x)
{
	int a = x;
	while (x != father[x])
		x = father[x];

	//路径压缩
	while (a != father[a])
	{
		int z = a;
		a = father[a];
		father[z] = x;
	}
	return x;
}

void Union(int a, int b)
{
	int fa = findfather(a);
	int fb = findfather(b);
	if (fa != fb)
		father[fa] = fb;
}

int main()
{

	int n, m;

	for (int i = 1; i <=n; i++)

		isroot[i] = false;
	}

	int tmp1, tmp2;
	for (int i = 0; i < m; i++)
	{
		cin >> tmp1 >> tmp2;
		Union(tmp1, tmp2);
	}

	for (int i = 1; i <= n; i++)
	{
		isroot[findfather(i)] = true;
	}
	
	int ans = 0;
	for (int i = 1; i <= n; i++)
	{
		ans += isroot[i];
	}

	system("pause");
	return 0;
}
