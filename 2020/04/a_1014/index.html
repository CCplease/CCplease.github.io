题目

样例

Input

2 2 7 5
1 2 6 4 3 534 2
3 4 5 6 7

Output

08:07
08:06
08:10
17:00
Sorry

思路和坑点

这道题读完，你就应该知道是一个线性结构，用vector(or 数组)，然后每一个窗口有很多的信息需要表示，那么结构体肯定跑不掉，结构体里包含一个starttime和endtime的变量，既然有排队，那么队列-queue肯定少不了，那么基本上框架就有了，一个一维的vector<结构体>就定下了，然后一个time存放每个人办理业务需要的时间，一个down_time存放每个人办完业务的时间，题目要求说有sorry的，所以个sorry数组存放当前这个人能否去选择好的窗口办理，所以本题需要的材料就准备好了，一个vector<结构体>，vector time,down_time,vector sorry，4个数组。
这题没什么难点，但是有如下几个复杂的点:

  • starttime和endtime分别代表的是，当前队列正在处理的人的时间点，以及当前队列最后一个人被服务完后的时间点
  • 判断当前这个人是否可以入队的条件是那个队列(窗口)的endtime是否大于等于1020，如果超出1020，表示当前这个人无法入队，放入sorry数组做记录。这也就意味着，得先判断是否可以入队，再去更新endtime
  • 因为down_time也是根据endtime来更新的，所以，down_time得再endtime更新之后
  • 队列q既可以存放当前这个即将入队(注意，是即将入队，因为不一定会入队成功)的人的编号或者是需要办理业务的时间，两个都可以，这里我选择前者

以上就是坑点，但其实这题不够严谨，因为当前某个人要入队办理，如果窗口时间大于等于1020则不予办理，认为这个人无法办理业务，但是现实中肯定会选择其他时间小于1020的窗口进行办理，所以你可以思考一下：如果当前即将入队的人无法入队，需要选择其他窗口该怎么办

#include<iostream>
#include<cstdio>
#include<vector>
#include<cstring>
#include<queue>
#include<sstream>
#include<algorithm>
#pragma warning(disable:4996)
using namespace std;

typedef struct node
{
	//这个地方我直接给480，也就是8点起开始计算，方便后面输出就不用×480了
	int starttime=480, endtime=480;

	//这里存放的是每个人离开的时间，依旧是从8点算，可以调试看一下值
	queue<int> q;
}Node;

int main()
{
#ifdef ONLINE_JUDGE                
#else    
	freopen("1.txt", "r", stdin);
#endif 
	
	int n, m, k, q;
	cin >> n >> m >> k >> q;

	vector<int> time(k + 1), down_time(k + 1);
	vector<bool> sorry(k + 1,false);

	for (int i = 1; i <= k; i++)
	{
		cin >> time[i];
	}

	vector<Node> window(n+1);
	int index = 1;
	for (int i = 1; i <= m; i++)
	{
		for (int j = 1; j <= n; j++)
		{
			if (index <= k)
			{
				window[j].q.push(index);

				//这个地方注意一下啊，我们先判断endtime是否大于等于1020，而不是endtime更新后在判断
				//因为判断他是否可以入队的条件是，在他入队前，窗口的endtime是否大于等于1020
				//如果大于540，说明接下来已经不能入队了
				if (window[j].endtime >= 1020)
					sorry[index] = true;

				window[j].endtime += time[index];

				if (i == 1)
					window[j].starttime += window[j].endtime;

				//给down-time存一下每个人离开的time,请注意，不能用
                //down_time[index]=window[j].starttime+time[index]
				//这个写法是错的，如果仅对这个样例好像是对的，但是仔细一想逻辑是错误的
				//想一下我们的starttime和endtime分别表示什么
				down_time[index] = window[j].endtime;
			
				index++;
			}	
		}
	}

	//现在处理黄线后面的
	while(index<=k)
	{
		//我们得看看每个柜台的starttime，挑出最小的，那就是先出队的，所以得用
		//两个变量保存当前柜台的编号及starttime
		int tempmin = window[1].starttime, tempwindow = 1;
		for (int j = 2; j <= n; j++)
		{
			if (tempmin > window[j].starttime)
			{
				tempmin = window[j].starttime;
				tempwindow = j;
			}
		}

		//到这里说明知道了最先应该出队的柜台
		window[tempwindow].q.pop();
		window[tempwindow].q.push(index);


		//这一步跟前面一样的，都要在endtime更新前判断，如果endtime已经是1020了，那么这个窗口就结束了
		//不会在有人入队了，所以可以直接用if-else减少时间，然后让这个index去其他窗口办理业务,可以减少
        //时间复杂度
		if (window[tempwindow].endtime >= 1020)
			sorry[index] = true;
		else
		{	//到这里说明新的人可以入队，所以要更新这个柜台的starttime,endtime和down_time
			window[tempwindow].endtime += time[index];
			window[tempwindow].starttime += time[window[tempwindow].q.front()];

			//这个地方注意，每个人的结束时间，就是窗口的end_time，所以一定要等当前窗口的end_time更新完
			//才可以队down_time进行更新，因为本身endtime就是表示当前窗口所有人处理完后的时间
			down_time[index] = window[tempwindow].endtime;
		}
		index++;
	}

	for (int i = 0; i < q; i++)
	{
		int num;
		cin >> num;
		if (sorry[num] == 1)
			cout << "Sorry" << endl;
		else
			printf("%02d:%02d\n", down_time[num] / 60, down_time[num] % 60);
	}

	system("pause");
	return 0;
}
